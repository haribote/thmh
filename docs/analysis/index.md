# analysis

Component discovery and prop / variant / token extraction. Feature IDs in this domain use the `ANA` prefix.

| ID | Title | Summary |
| --- | --- | --- |
| [ANA001](ANA001_component-discovery.md) | Component discovery | Which exported declarations in a file are worth documenting. |
| [ANA002](ANA002_react-props-adapter.md) | React props adapter | Props, types, and JSDoc read off the first parameter through the type checker. |
| [ANA003](ANA003_cva-variant-adapter.md) | cva variant adapter | `cva()` calls read as base classes, axes, defaults, and compound variants. |
| [ANA004](ANA004_variant-matrix.md) | Variant matrix | Axes derived from a variant definition, and every combination of their options. |
| [ANA005](ANA005_manifest-assembly.md) | Manifest assembly | The stages joined into one record per component, with failures isolated. |

See [requirements](../requirements.md) and [CONTRIBUTING.md](../../CONTRIBUTING.md).
