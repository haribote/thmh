# manifest

The `catalog.json` contract: schema, generation, and versioning. Feature IDs in this domain use the `MAN` prefix.

| ID | Title | Summary |
| --- | --- | --- |
| [MAN001](MAN001_catalog-schema.md) | Catalog schema | The shape of `catalog.json`, and the contract every consumer reads. |
| [MAN002](MAN002_dev-manifest-refresh.md) | Dev-time manifest refresh | One catalog held in memory, re-derived on change and pushed to subscribers. |
| [MAN003](MAN003_catalog-generation.md) | Catalog generation | A project directory turned into one catalog, in a single pass. |

See [requirements](../requirements.md) and [CONTRIBUTING.md](../../CONTRIBUTING.md).
