Follow `docs/tdd.md` — the canonical, agent-agnostic TDD procedure (Red → Green → Refactor, one test at a time). It is the single source of truth; this skill is a thin adapter that points to it.

Read that document and follow its cycle, non-negotiable rules, and test commands. Present the test list and align with the user before implementing, and base any "tests pass" claim on an actual test run.
